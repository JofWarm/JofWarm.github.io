浏览器原理学习

发表于2025-10-02|更新于2025-10-06

浏览器的基本功能

浏览器将用户输入的web资源以图形化的形式展现出来，一般资源格式为HTML。用户用URI(Uniform Resource Identifiler 统一资源标识符)来指定所请求资源的位置。

  • URI(Uniform Resource Identifiler 统一资源标识符)
  • URL（uniform resource location）统一资源定位符（或统一资源定位器）
  • URN(uniform resource name)统一资源命名

注释：URL是URI的一种，是URI命名机制的一个子集

标签页进程

  • 浏览器主进程：负责页面显示、用户交互、子进程管理、同时提供存储等功能
  • 渲染进程：核心任务是将HTML、CSS、JavaScript转化为用户可以与之交互的页面，chrome默认为每一个标签页开启一个渲染进程
  • GPU进程：用于支持CSS的3D效果、图形、视频渲染
  • 网络进程：主要负责页面的网络资源加载
  • 插件进程：负责浏览器插件运行

浏览器结构

用户界面

除了窗口显示页面，其他用户看见的都属于用户界面，是用户与浏览器本身发生交互的功能组件。

浏览器引擎

用户界面和渲染引擎之间传送指令，是渲染引擎的一个接口。
功能包括：

  • 加载指定的URI，实现前进/后退/刷新等功能
  • 查询/修改渲染引擎的设置
  • 在用户界面和呈现引擎之间传送指令

渲染引擎

负责指定的URI生成可视化界面，解析HTML、CSS以及图片等资源。
市面上常见五种内核：

  • Trident：IE,360(1.0-5.0),猎豹浏览器
  • Gecko：firefox浏览器
  • Presto：Opera浏览器早期使用
  • Webkit：谷歌浏览器,Safari，手机浏览器
  • Blink：谷歌、Opera共同开发

网络

实现http/https，FTP等文件传输协议，可以在不同字符集之间转换，也可以缓存最近检索的资源。

浏览器缓存机制主要是 HTTP缓存。

显示后端

用于绘制基本的窗口小部件。

JS解释器

用于解析和执行JS代码，在ES6之前，渲染引擎和JS解释器是在一起工作的。
常见问题：

  • 事件循环
  • JS垃圾回收机制
  • V8工作原理

数据持久层

浏览器需要在硬盘上保存各种数据，例如LocalStorage，SessionStorage，Cookie，安全证书，缓存等低级数据。

浏览器的多进程架构

  • 浏览器主进程：负责页面显示、用户交互、子进程管理、同时提供存储等功能
  • 渲染进程：核心任务是将HTML、CSS、JavaScript转化为用户可以与之交互的页面，chrome默认为每一个标签页开启一个渲染进程
  • GPU进程：用于支持CSS的3D效果、图形、视频渲染
  • 网络进程：主要负责页面的网络资源加载
  • 插件进程：负责浏览器插件运行

浏览器的主要结构

用户界面

除了窗口显示页面，其他用户看见的都属于用户界面，是用户与浏览器本身发生交互的功能组件。

浏览器引擎

  • 用户界面和渲染引擎之间传送指令，是渲染引擎的一个接口
  • 用来加载指定的URI，实现前进/后退/刷新等功能
  • 用来查询/修改渲染引擎的设置
  • 在用户界面和呈现引擎之间传送指令

渲染引擎

  • 负责指定的URI生成可视化界面，解析HTML、CSS以及图片等资源
  • 常见渲染内核：Trident、Gecko、Presto、Webkit、Blink

网络

  • 实现http/https，FTP等文件传输协议
  • 可以在不同字符集之间转换
  • 可以缓存最近检索的资源
  • 浏览器缓存机制主要是 HTTP缓存

JS解释器

  • 解析和执行JS代码
  • ES6之前，渲染引擎和JS解释器合并工作
  • 常见问题：事件循环、垃圾回收、V8工作原理

数据持久层

  • 保存浏览器在硬盘上的数据，包括LocalStorage、SessionStorage、Cookie、安全证书、缓存等

浏览器的请求与缓存机制

URL检测

  • 浏览器自动判断输入的是查询内容还是正常的URL
  • 文本内容直接拼接成URL请求
  • 符合URL规则直接按URL访问

查找缓存

  • 浏览器先在数据存储中查询是否有需要请求的文件
  • 如果没有缓存，第一次请求进入网络请求过程
  • 如果有缓存，浏览器拦截请求、返回缓存、判断文件是否过期

强制缓存与协商缓存

  • 强制缓存：浏览器自导自演，未过期直接使用缓存，不访问服务器
  • 协商缓存：浏览器发现文件过期，需要与服务器通信，判断文件是否更新

判断资源是否过期

  • Response Header包含 Expires 和 Cache-Control
  • Cache-Control中包含 max-age（可缓存时间）
  • Expires是HTTP1.0产物，Cache-Control是HTTP1.1产物

判断资源是否更新

  • Response Header中包含 Last-Modified 和 ETag
  • Last-Modified记录文件最后修改时间
  • ETag记录文件内容变化，相当于版本号

DNS域名解析过程

  • 浏览器缓存 → 系统hosts文件 → 本地DNS解析器缓存 → 本地域名服务器 → 根域名服务器 → 主域名服务器 → 下一级域名服务器

递归查询

  • 本地域名服务器不知IP地址时，替主机继续查询
  • 主机向本地域名服务器递归查询
  • 本地域名服务器再向根域名服务器递归查询
  • 顶级域名服务器继续递归，直到权限域名服务器返回结果

迭代查询

  • 根域名服务器返回下一步应查询的服务器地址
  • 本地域名服务器依次向各级服务器迭代查询
  • 最终得到域名对应IP

建立TCP连接

TCP与UDP区别

  • TCP/IP是互联网协议族，包括TCP、UDP、IP、FTP、HTTP、ICMP、SMTP等
  • 协议四层：链路层、网络层、传输层、应用层
  • UDP无连接、面向报文、不可靠、头部开销小
  • TCP可靠、面向连接、数据分段重组

TCP断开连接

  • 第一次挥手：客户端发送FIN，进入FIN_WAIT_1
  • 第二次挥手：服务端回ACK，客户端进入FIN_WAIT_2
  • 第三次挥手：服务端发送FIN，请求关闭，进入LAST_ACK
  • 第四次挥手：客户端发送ACK，进入TIME_WAIT

高频问答

  • 三次握手：保证可靠双工通信、同步序列号、防止资源浪费
  • 四次挥手：全双工，每方向单独关闭
  • TIME_WAIT：确保最后ACK到达被动关闭方

浏览器解析HTML并布局渲染

  • 解析HTML生成DOM树
  • 解析CSS生成CSS规则树
  • DOM树和CSS规则树生成渲染树

渲染进程线程

  • GUI渲染线程：解析HTML、CSS，构建DOM树，布局和绘制
  • JS引擎线程：解析执行JS代码，与GUI线程互斥
  • 事件触发线程：处理事件轮询
  • 定时触发器线程：定时器相关
  • 异步HTTP请求线程：处理AJAX请求

JS解析引擎

  • 解析JavaScript并执行
  • 解析过程：词法分析 → 语法分析 → 生成语法树 → 执行代码
  • 执行过程：
    • 创建执行上下文（变量对象、作用域链、this）
    • 初始化阶段（变量声明、函数声明、参数）
    • 执行阶段（逐条执行代码）

异步任务机制

  • 宏任务：整体script、setTimeout、UI事件、I/O等
  • 微任务：Promise回调、MutationObserver
  • 事件循环规则：宏任务执行时产生微任务，宏任务执行完立即执行微任务，再渲染

回流和重绘解析

重绘（Repaint）

  • 绘制（paint）：第一次渲染内容称之为
  • 之后重新渲染称之为重绘
  • 重绘不会带来重新布局所以并不一定伴随重排
  • 导致重绘的属性：color background box-shadow…

回流（reflow）

  • 也称之为重排
  • 布局（layout）：第一次确定节点大小和位置
  • 回流：对节点大小、位置修改重新计算
  • 回流必然导致重绘
  • 所以回流是一件很消耗性能的事情
  • 导致回流的属性：width top position…

常见触发“回流/重排”操作

  • 因为reflow成本高于repaint
  • 因此开发中尽量避免回流：
    • 将多次改变样式属性的操作合并成一次操作
    • 预先定义好class，然后修改DOM的className
    • 尽量避免频繁的操作DOM
      • 可以在一个父元素中将要操作的DOM操作完成在一次性操作
    • 尽量避免通过getComputedStyle获取尺寸、信息等位置
    • 对某些元素使用position的absolute或者fixed
      • 这样开销相对较小

合成和性能优化

  • 绘制的过程，可以将布局后的元素绘制到多个合成图层中
    • 这是浏览器的一种优化手段
  • 默认情况下，标准流中的内容都是被绘制在同一个图层（Layer）中的
  • 而一些特殊的属性，会创建一个新的合成层（ CompositingLayer ），并且新的图层可以利用 GPU 来加速绘制
    • 因为每个合成层都是单独渲染的
  • 常见的一些属性：
    • 3D transforms
    • video、 canvas、 iframe
    • opacity 动画转换时
    • position: fixed
    • animation 或 transition 设置了 opacity、 transform
  • 分层确实可以提高性能，但是它以内存管理为代价，因此不应作为 web 性能优化策略的一部分过度使用
