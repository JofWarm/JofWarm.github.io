Promise 学习与应用详解

发表于2025-09-26|更新于2025-10-02

Promise 的两个核心特点

  1. 对象的状态不受外界影响
    Promise 对象代表一个异步操作，有三种状态：

    • pending（进行中）
    • fulfilled（已成功）
    • rejected（已失败）

    只有异步操作的结果，可以决定当前是哪一种状态，任何其他操作都无法改变这个状态。

  2. 状态一旦改变，就不会再变
    Promise 对象的状态只能从：

    • pending → fulfilled
    • pending → rejected

    一旦发生变化，状态就凝固了（称为 resolved 已定型），不会再变。
    即使在改变之后再添加回调函数，也会立即得到结果。

Promise 链式调用

但是，上面的链式调用只是简单的格式化处理，并没有彻底解决回调地狱问题。
真正要解决异步编程的可读性与可维护性问题，一定要结合 async/await 来实现“异步写同步”。

async/await 结合 Promise

async/await 可以让异步代码更加直观，避免层层嵌套的回调，提升代码可读性。

Promise 常用 API

  • Promise 构造函数：Promise (executor) {}
    • executor：执行器函数 (resolve, reject) => {}
    • resolve：内部定义成功时调用的函数 value => {}
    • reject：内部定义失败时调用的函数 reason => {}
