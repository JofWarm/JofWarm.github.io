JF CLI 脚手架问题与解决总结

1. Vite 启动后无法访问 localhost

问题描述

启动 Vite 服务后控制台显示成功，网址为 http://localhost:3000/，但浏览器访问时提示找不到页面。

解决方案

  • 确认本地防火墙或杀毒软件未阻止端口。
  • 使用 --host 参数启动，允许外部访问。
  • 确认项目目录下存在 index.html，且其位置正确（一般在项目根目录或 src 目录）。
  • 检查 vite.config.js 中 server.host 是否正确配置。

2. Vue 组件中无法正确解析路径别名 @

问题描述

导入资源如 @/assets/login.jpg 报错，提示找不到文件。

解决方案

  • 在 vite.config.js 中配置路径别名：
  • 确认资源文件存在于正确路径。

3. 依赖包未安装导致模块找不到

问题描述

引入第三方库如 axios、vue-router 报错，提示模块找不到。

解决方案

  • 在脚手架项目模板中集成自动安装依赖的功能，确保用户创建项目后依赖自动安装，无需手动执行安装命令。
  • 例如，在脚手架代码中使用 execa 或 child_process 调用 npm install 或 yarn install 自动安装依赖。
  • 这样可以避免因缺少依赖导致的模块找不到错误，提高用户体验。
